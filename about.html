Project Story

About the project

This project explores the shift from casual prompting to intentional AI system design.

Inspiration

The idea came from noticing that many people use AI by typing one-off prompts, but far fewer understand how to design the structure around those prompts. We wanted to show that AI literacy is not just about asking better questions — it is also about building better instruction systems.

What it does

From Prompt User to System Architect is an interactive web project that transforms a basic user prompt into three stages of maturity: a raw prompt, a refined prompt, and a system-level workflow prompt. It also scores the prompt and provides reusable outputs.

How it was built

Version 2 was built with HTML, CSS, and JavaScript as a front-end concept prototype. The site includes a prompt transformation engine, a scoring interface, prompt templates, copy/export tools, and a polished minimal product-style UI.

Challenges

One challenge was turning an abstract concept like “system architecture” into an interactive experience that still feels simple and approachable. Another was balancing educational clarity with a polished product feel so the project could function both as a demo and a learning tool.

What was learned

The project reinforced that strong AI results come from role design, task scoping, context, formatting rules, and quality control — not just from rewriting a prompt. It also highlighted the value of designing AI workflows in a way that users can understand visually.

What’s next

The next version would connect to the OpenAI API so the transformation is generated live. It could also add saved prompt libraries, prompt analytics, side-by-side comparisons, and guided teaching modules for learning prompt architecture in real time.

Prompt maturity ladder

01

Prompt User

Asks for an output directly with little structure or context.

02

Prompt Engineer

Adds context, audience, formatting, and clearer instructions to improve the response.

03

System Architect

Designs the AI’s role, workflow, constraints, and quality checks as a full interaction system.
